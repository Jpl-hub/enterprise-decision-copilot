# 模型路线图

## 为什么要接深度学习

比赛里“用了 AI”不等于有竞争力，关键是它是否真正提升了：

- 数据抽取效率
- 证据检索质量
- 风险识别能力
- 报告生成质量

## 高奖版本应采用的四条模型线

### 1. 大模型抽取与生成

- 财报 PDF -> 关键指标结构化
- 研报摘要 -> 观点提炼
- 用户问题 -> 企业诊断与报告生成
- 输出时必须附带来源追溯、证据片段和推理链摘要

### 2. 多模态文档理解

- 年报 PDF 中的表格、图表、章节结构并不稳定
- 仅靠纯文本正则容易误抓季度表、分部表、管理层说明
- 应引入“版面感知 + 视觉理解”链路，用于识别：
  - 主要会计数据表
  - 合并资产负债表
  - 合并利润表
  - 经营数据图表

建议落地方向：

- 第一阶段：`PyPDF2 + 规则 + LLM`，已完成基础版
- 第二阶段：接入 OCR / 表格解析
- 第三阶段：接入视觉语言模型，用于复杂表格与图表理解

### 3. 文本检索与语义召回

- 研报切片
- 嵌入向量或 API Embedding
- 语义召回 + 证据追溯
- 重排模型用于提高问答和报告生成的证据质量

### 4. 财务时序风险预测

- 输入：企业连续多年财务指标
- 输出：下一期风险等级或风险分值
- 方案：LightGBM 基线 + LSTM/TFT 深度时序模型

这里要强调一个比赛叙事：

- 不是让深度学习替代财务分析
- 而是让深度学习从历史面板中学习“风险演化模式”
- 再把结果交给智能体做解释、归因和建议生成

## 当前仓库中的深度学习入口

- `scripts/extract_financial_metrics_with_llm.py`
- `scripts/train_risk_sequence_model.py`

## 冲高奖的推荐组合

真正适合答辩展示的组合不是“所有模型都上”，而是下面这套：

1. 文档理解：多模态抽取年报核心表格与图表
2. 检索增强：研报与财报证据召回 + 重排
3. 智能体：多工具调用完成问数、对比、诊断、报告生成
4. 时序预测：对企业经营风险和经营韧性做趋势预测

## 暂不建议一开始就做的事

- 一上来做行业专属大模型微调
- 一上来做复杂 RL 或长期自主学习
- 一上来做多智能体群体协作并行推演

原因很简单：

- 当前比赛更吃“真实数据闭环 + 业务价值 + 可解释性”
- 这些方向成本高，短期不一定带来更高展示效果

## 数据规模起来后的补全项

等真实结构化数据达到规模后，我会继续把这条线补完整，包括：

- 训练集构造
- 指标评估
- 消融实验
- 模型对比
- 多模态抽取与纯文本抽取效果对比
- 召回前后报告质量对比
