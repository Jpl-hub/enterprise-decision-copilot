# 企智策源比赛级产品蓝图

## 1. 项目定位

这是一个面向企业运营分析与决策支持的比赛项目，不是普通后台，也不是单纯聊天机器人。

产品必须同时体现三条主线：

1. 传统应用 + Agent
   目标是把“企业分析、企业对比、数据治理、材料导出”做成真实可用的业务系统。
2. 深度学习 / 大模型 / 多模态
   目标是把研报检索、财报图表抽取、风险预测、多模态证据引用接进分析链。
3. 大数据底座
   目标是把湖仓、质量治理、批处理作业、后续 Spark 扩展讲清楚并落进系统。

## 2. 产品总原则

1. 首页只负责一件事：承接问答与指挥全局。
   首页是唯一主问答入口，必须一眼看懂、立刻可问、可跳转。
2. 企业分析页只负责看结果，不再重复问答。
   企业页是报告页、证据页、风险页，不是第二个首页。
3. 每个页面只有一个主动作。
   不允许同屏出现两个相同职责的输入区、两个主 CTA、两套重复导航。
4. 文案只保留对决策有帮助的内容。
   禁止为了“显得丰富”堆砌灰色小字、装饰性标签、无行动价值的解释。
5. 比赛展示必须抓眼，但视觉服务于结构，不服务于堆砌。
   允许大屏感、图表感、科技感，但必须和证据链、时效链、分析链对应。

## 3. 信息架构

### 3.1 首页

角色：
- 唯一问答入口
- 全局态势总览
- 企业与任务切换入口

必须保留：
- 企业选择
- 主问答区
- 4 个以内的主跳转入口
- 1 组高价值可视化
- 风险 / 行业 / 宏观的少量关键卡片

必须删除：
- 与用户动作无关的说明文字
- 重复引导语
- 超过 1 层的解释性小字
- 为了凑满页面而存在的静态卡片

### 3.2 企业分析页

角色：
- 企业级结果页
- 财报、研报、风险、多模态证据的聚合页

必须保留：
- 企业头图与核心结论
- 经营分析
- 风险画像
- 证据资料
- 回到首页继续追问

必须删除：
- 第二套问答区
- 第二套任务流
- 与首页重复的输入交互

### 3.3 企业对比页

角色：
- 横向对比与证据验证

必须保留：
- 对比结论
- 对比矩阵
- 证据流
- 财报图表锚点

### 3.4 数据治理页

角色：
- 数据底座、质量、系统能力展示页

必须保留：
- 湖仓分层
- 披露覆盖
- 多模态覆盖
- 系统三主线 readiness

## 4. 视觉原则

参考方向不是“普通国产后台”，而是：
- 苹果式简约秩序
- 比赛大屏的信息聚焦
- 科技感但克制的深色驾驶舱

具体要求：

1. 标题层级必须少。
   一屏内只允许 1 个大标题，2 到 3 个区块标题。
2. 小标签能删就删。
   section tag 不是默认项，必须证明它能帮助理解，否则删除。
3. 说明文必须压缩。
   一张卡片最多一句说明，不允许三段式灰字说明。
4. 按钮必须成组对齐。
   同组按钮高度一致、边距一致、主次明确。
5. 图表必须可解释、可点击。
   不是装饰图，而是能导向企业、数据治理或证据页。

## 5. 创新点落位

### 5.1 传统应用 + Agent
- 首页唯一主问答区
- 线程记忆
- 企业焦点继承
- 分析 / 对比 / 质量 / 导出闭环

### 5.2 深度学习 / 多模态
- 研报语义检索
- 风险模型
- 财报图表多模态抽取
- 页锚点回链
- SFT 数据集沉淀

### 5.3 大数据
- Bronze / Silver / Gold / Mart
- DuckDB + Parquet
- Spark-ready 作业规划
- 数据时效和质量治理看板

## 6. 当前页面整改顺序

1. 首页
   先减小灰字密度，控制卡片数量，固定唯一问答区。
2. 企业分析页
   去掉第二问答区，只保留结果与证据。
3. 对比页
   强化图表与证据，不再叠解释。
4. 数据治理页
   保持系统视角，避免业务页内容混入。

## 7. 后续实现约束

后续任何页面改动，如果违反以下任一条，不允许继续堆：

1. 是否造成重复入口。
2. 是否增加无用说明文字。
3. 是否破坏页面主次顺序。
4. 是否让页面从“系统”退回“功能堆砌”。
5. 是否不能直接支撑比赛讲解。
