# 智能架构详细设计 2026

## 1. 智能层定位

智能层负责把用户问题转成可执行、可验证、可交付的决策流程。

它的目标不是“多 agent 越多越好”，而是：

1. 问题理解准确
2. 证据调用正确
3. 过程可解释
4. 结论可验证
5. 输出可动作化

## 2. 智能层总体结构

智能层由六部分组成：

1. 主代理 Coordinator
2. 领域 Skills
3. 检索增强 RAG
4. 记忆系统
5. 决策验证器
6. 输出组装器

## 3. 主代理

### 3.1 职责

1. 识别用户意图
2. 绑定上下文对象
3. 选择执行链
4. 调用领域 skills
5. 汇总中间结果
6. 触发验证
7. 产出最终响应

### 3.2 执行原则

- 默认单主代理负责协调
- skill 按需调用
- 避免多个 agent 常驻对话带来冗余复杂度

## 4. 领域 skill 体系

### 4.1 财报解析 skill

负责：

- 年报 / 季报结构化提取
- 核心财务指标归一
- 章节重点抽取

### 4.2 研报抽取 skill

负责：

- 券商观点提取
- 情绪倾向识别
- 评级与主题提取

### 4.3 证据对齐 skill

负责：

- 财报与研报对齐
- 指标和文字证据绑定
- 时间维度统一

### 4.4 风险穿透 skill

负责：

- 财务风险
- 经营风险
- 行业风险
- 数据缺口风险

### 4.5 同业对比 skill

负责：

- 企业指标横向比较
- 证据差异归纳
- 推荐结论生成

### 4.6 行业趋势 skill

负责：

- 行业主题识别
- 外部变量提取
- 企业映射

### 4.7 数据质量诊断 skill

负责：

- 覆盖度检查
- 时效性检查
- 异常数据识别
- 风险提示

### 4.8 管理动作生成 skill

负责：

- 从分析结果转成动作板
- 定义优先级
- 给出观察指标和触发条件

## 5. 检索增强

### 5.1 检索目标

- 找到真实证据
- 找到最相关证据
- 找到最新证据

### 5.2 检索对象

1. 财报切片
2. 研报切片
3. 行业资料切片
4. 宏观数据切片
5. 历史线程记忆

### 5.3 检索流程

1. Query 重写
2. 多路检索
3. 重排序
4. 证据去重
5. 证据打分
6. 结果回链

## 6. 记忆系统

### 6.1 线程记忆

保存：

- 当前对话结论
- 关键证据
- 下一步追问

### 6.2 公司记忆

保存：

- 公司长期经营画像
- 历史重点风险
- 关键事件轨迹

### 6.3 行业记忆

保存：

- 行业长期主题
- 常见风险与机会
- 关键外部变量

### 6.4 项目记忆

保存：

- 当前项目推进状态
- 输出过的结论
- 已交付内容

## 7. 决策验证器

### 7.1 目标

不是让模型“再想一遍”，而是做独立验证。

### 7.2 检查项

1. 证据是否真实
2. 结论是否可回链
3. 风险是否遗漏
4. 推理是否跳步
5. 是否存在自相矛盾
6. 是否满足发布门禁

## 8. 深度学习模块

### 8.1 语义编码

用于：

- 文档检索
- 语义相似匹配
- 聚类和主题识别

### 8.2 排序模型

用于：

- 检索重排序
- 证据优先级排序

### 8.3 分类模型

用于：

- 文档分类
- 任务分类
- 情绪与观点分类

### 8.4 风险评分模型

用于：

- 风险概率计算
- 高风险信号识别

## 9. 输出组装

输出不能只是一段话，必须结构化：

1. 结论摘要
2. 关键论点
3. 证据清单
4. 风险提示
5. 下一步动作
6. 推荐追问

1. 领域 skill protocol

3. 决策验证器
