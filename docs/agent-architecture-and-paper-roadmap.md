# Agent 架构与论文路线

## 1. 当前判断

这个项目的主线必须是：

- 主体：任务型 Agent 产品
- 承载：少量但扎实的传统应用界面
- 核心加分：深度学习 / 大模型 / 多模态 / 大数据工程
- 展示增强：后续再接数字人、Unity、演示舱

也就是说，系统不能做成：

- 纯 Web 后台
- 纯聊天机器人
- 纯 API 包装器
- 纯多智能体概念展示

## 2. 从 MiroFish 借鉴什么

参考仓库：
- https://github.com/666ghj/MiroFish

从仓库公开 README 可以提炼出的有效思路：

1. 先讲“用户完成什么任务”，再讲技术。
2. 系统不是单轮问答，而是完整工作流。
3. Agent 的价值来自：记忆、步骤、阶段产物、可持续互动。
4. 高级系统感来自“数字沙盘/驾驶台/世界状态”，不是来自堆按钮。
5. 报告生成必须是主产物之一，而不是附属功能。

这些思路适合迁移到本项目。

## 3. 不该照抄什么

MiroFish 面向的是“群体模拟/舆情预测/数字世界推演”，与本题不完全一致。

本项目不应该直接照抄：

1. 超大规模群体仿真叙事
- 本题核心不是社会模拟，而是企业运营分析与决策支持。

2. 为了“多智能体”而多智能体
- 如果没有清晰的任务分工，只会让系统变成复杂演示而不是好产品。

3. 先上图谱和超大仿真，再补业务
- 本题要先交付企业分析闭环，再扩展复杂能力。

## 4. 本项目 2026 年不落后的 Agent 路线

结合 LangGraph / Deep Agents / Memory 官方文档，可以抽出适合本项目的路线：

1. 持久化工作流 Agent
- 线程持久化
- 状态持久化
- 任务模式切换
- 计划步骤可视化
- 阶段产物可视化

2. 任务代理层
- 企业诊断代理
- 风险预警代理
- 决策建议代理
- 行业趋势代理
- 数据治理代理

3. 子代理 / 专项分析器
- 多模态财报抽取子代理
- 风险模型解释子代理
- 证据召回子代理
- 报告生成子代理

4. 人机协同闭环
- 数据异常进入人工复核
- 关键分析允许人工确认
- 导出前给出可信度提示

这条路线比“纯聊天”更强，也比“空壳多智能体”更稳。

- 任务状态机
- 计划步骤
- 任务产物
- 线程记忆
- 审计日志

### 5.3 AI 层

- 决策大模型：推理、总结、问答、任务编排
- 多模态模型：财报图表/表格补全
- 风险预测模型：分类/评分/解释

### 5.4 大数据层

- 多源异构采集

- 数据质量规则

- 可追溯证据链

## 6. 深度学习 / 大模型怎么真正引入

不能只是说“用了大模型”，而要形成三条真正的 AI 主链路：

### 6.1 企业分析主链路

- 输入：用户问题 + 企业对象
- 模型：大模型做任务理解、证据整合、结论生成
- 输出：结论、动作建议、证据引用、追问建议

### 6.2 多模态抽取主链路

- 输入：PDF 财报页面图
- 模型：视觉语言模型抽取图表与表格字段
- 输出：结构化字段 + 页面来源 + 抽取说明

### 6.3 风险预测主链路

- 输入：财务特征、趋势特征、研报信号
- 模型：风险预测模型 / 深度学习模型
- 输出：风险概率、风险等级、关键因子

这三条链路必须在页面里被用户看到，而不是只写在 README。

## 7. 论文路线

这个项目最终完全可以产出一篇高质量论文，建议方向：

题目候选：
- 《面向企业运营分析与决策支持的任务型智能体系统设计与实现》
- 《融合多模态财报抽取与风险预测的企业决策智能体研究》
- 《结合大数据治理与大模型编排的企业分析智能体系统》

### 7.1 论文创新点

1. 任务型企业分析智能体架构
- 不是单轮问答，而是任务模式、步骤、产物、线程、审计的完整闭环。

2. 多模态财报证据补全
- 对复杂 PDF 图表/表格进行视觉抽取，进入企业分析证据链。

3. 风险模型 + Agent 协同
- 规则、模型、大模型协同，而不是单模型硬答。

4. 数据治理闭环
- 数据覆盖、异常检测、复核工单、可信度展示一体化。

### 7.2 论文结构

1. 引言
2. 相关工作
3. 系统总体架构
4. 任务型 Agent 设计
5. 多模态与风险模型设计
6. 大数据治理与质量闭环
7. 实验与案例分析
8. 结论与展望

### 7.3 实验设计

- 任务完成度对比
- 风险判断质量对比
- 多模态字段补全效果
- 证据可追溯性评估
- 用户操作链路效率评估

- 证据展开

2. 给每种任务模式做差异化输出

4. 开始准备论文材料
