# 产品页面详细设计 2026

## 1. 产品面总原则

产品面只保留用户真正使用的页面，不再把控制面、治理面、展示面混在一起。

产品面页面：

1. 开始分析 / 企业问答中枢
2. 数字人聊天室
3. 企业分析工作台
4. 企业对比判断
5. 导出与交付

系统面页面：

1. 系统控制面
2. 数据治理中心
3. 审计与质量页

## 2. 开始分析 / 企业问答中枢

### 2.1 页面目标

这是主入口页面，承担两个职责：

1. 选择当前分析对象
2. 围绕单个问题进行专业问答

### 2.2 页面结构

页面结构应为：

1. 顶部轻量 Hero
2. 企业选择区
3. 问答主区域
4. 历史线程抽屉
5. 薄推理状态条

### 2.3 页面规则

- 主区域必须是聊天
- 聊天区域固定高度
- 输入框固定在底部
- 历史线程默认折叠
- 推理过程默认简洁
- 一屏优先看到对话

### 2.4 绝对禁止

- 右侧再堆一整列说明卡
- 页面中部再放四到六个大入口卡
- 把工作台、会议室、控制面都塞进这个页面
- 让聊天区无限向下增长

## 3. 数字人聊天室

### 3.1 页面目标

数字人聊天室是独立产品模块，承担：

1. 活体陪聊
2. 轻松问答
3. 场景化互动
4. 路演彩排
5. 后续实时语音 / 视频 / WebSocket 接入

### 3.2 页面结构

1. 数字人舞台
2. 场景切换区
3. 聊天主区域
4. prompts 与快捷问题

### 3.3 页面特征

- 比企业问答更有角色感
- 比企业问答更有互动感
- 支持未来接娱乐向场景
- 不承载企业深度分析报告

## 4. 企业分析工作台

### 4.1 页面目标

工作台承担单企业深度分析，不承担主聊天任务。

### 4.2 页面结构

1. 企业身份区
2. 核心判断区
3. 证据锚点区
4. 风险穿透区
5. 经营拆解区
6. 图表与多模态证据区

### 4.3 设计重点

- 强证据回链
- 强报告感
- 强专业输出
- 弱聊天感

## 5. 企业对比判断

### 5.1 页面目标

企业对比页负责横向比较，不负责细节深钻。

### 5.2 页面结构

1. 对比对象选择
2. 关键指标对比
3. 风险差异对比
4. 证据差异对比
5. 推荐结论与动作

## 6. 导出与交付

### 6.1 页面目标

把中间分析过程变成正式交付物。

### 6.2 输出类型

1. 报告摘要
2. 证据清单
3. 路演版材料
4. 答辩版材料
5. 管理动作清单

## 7. 系统控制面

### 7.1 页面目标

系统控制面不是用户产品页，它是 control plane。

### 7.2 面向角色

1. 评委
2. 项目负责人
3. 管理员

### 7.3 页面结构

1. 控制面定位说明
2. 发布门禁
3. 五条主线状态
4. 当前阻塞项
5. 下一步推进动作
6. 架构全景

## 8. 交互原则

### 8.1 所有聊天页

- 输入框固定
- 历史消息滚动
- 支持快捷追问
- 不做长页面

### 8.2 所有分析页

- 先结论，后证据
- 先摘要，后细节
- 图文混排
- 引用清晰

### 8.3 所有系统页

- 先定位，再状态，再阻塞，再动作
- 避免和用户页面混淆

## 9. 页面优先级

第一优先级：

1. 企业问答中枢
2. 数字人聊天室
3. 系统控制面

第二优先级：

1. 企业分析工作台
2. 企业对比判断
3. 导出与交付

## 10. 最终要求

产品面必须体现：

- 专业
- 清晰
- 可展示
- 不堆模块
- 不滥用卡片

所有后续前端迭代都必须以本文件为准。
