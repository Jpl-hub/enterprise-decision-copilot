# 需求分析

## 1. 赛题要求拆解

根据官方赛题《2026年（第19届）中国大学生计算机设计大赛大数据主题赛“智能体赋能的企业运营分析与决策支持系统”赛题说明》，本题的核心要求是：

1. 以智能体为核心载体，而不是单纯的数据可视化系统。
2. 必须围绕一个具体领域做深做透，赛题明确给了医药生物、先进制造、新能源等行业方向。
3. 必须整合多源真实数据，至少覆盖财报、研报、宏观数据三类。
4. 必须支持自然语言交互、企业评估、风险机会洞察、定制化内容生成。
5. 需要做到信息来源追溯、归因分析和逻辑拆解，提升决策可信度。
6. 系统要能在指定或推荐平台中完成演示，源码、部署包、视频和设计报告都要形成闭环。

## 2. 高分作品必须具备的四层能力

如果想冲高奖，系统不能只做某一层，必须四层同时成立：

1. 智能体层：会理解问题、选择工具、拆解任务、返回轨迹。
2. 传统应用层：有正式 Web 系统、接口、数据库、可演示工作台。
3. 大数据层：有真实多源数据、批处理脚本、数据治理、后续分布式扩展能力。
4. AI/深度学习层：有大模型能力、语义检索能力、风险预测能力，多模态能力可继续扩展。

## 3. 赛题给出的目标用户

赛题原文实际上已经暗示了三个高分用户场景：

1. 投资者：自然语言问数、企业对比、价值线索发现。
2. 企业管理者：经营诊断、行业对标、战略优化建议。
3. 监管/风控角色：自动化风险筛查、异常监测、预警提示。

我们的系统展示重点优先放在“企业管理者 + 投资研究 + 风控预警”三位一体，这样更容易在答辩中体现业务价值和技术深度。

## 4. 高分逻辑

真正决定上限的是：

- 数据链路是否真实完整
- 官方来源是否可追溯、可复核、可解释
- 行业建模是否有针对性
- Agent 是否能调用分析能力而不是只会聊天
- 系统是否同时具备诊断、报告、决策、预测四类能力
- 结论是否有来源追溯和指标支撑
- 页面、视频、答辩链路是否形成闭环

## 5. 官方数据范围与合规边界

官方数据路径文件明确给了三类公开来源：

- 上海证券交易所、深圳证券交易所、北京证券交易所的定期报告
- 东方财富公开研报页面
- 国家统计局公开宏观数据页面

因此我们的数据策略必须遵循：

1. 以官方给定链接为主，不自造数据。
2. 不使用来源不明、涉密、违规数据。
3. 不对政府或交易所站点进行高频、破坏性抓取。
4. 所有分析结果都要能回溯到原始披露文件或公开页面。

## 6. 为什么先不用微调

当前阶段微调不是必须项，原因是：

- 还没有高质量、稳定、足够规模的行业监督数据集
- 比赛更容易被看到的是完整度、创新点、可解释性与工程实现
- 先把真实数据、检索增强、指标体系、Agent 工作流和系统闭环做扎实，收益更高

## 7. 项目定位

第一版默认聚焦医药生物行业，原因：

- 公开财报与研报样本丰富
- 宏观指标与行业叙事更容易建立联系
- 研发、盈利、现金流、政策敏感性都比较强，适合做多维诊断
- 很适合同时做企业运营分析、投资研究和风险预警

系统设计保持行业可切换，后续可以迁移到新能源、电子信息等赛道。
