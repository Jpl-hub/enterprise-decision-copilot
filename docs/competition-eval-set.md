# 赛题评测样例集

## 1. 目的

这份样例集专门服务当前赛题，不是通用聊天基准。

每次修改前端编排、线程上下文、skill 路由或分析文案后，都要至少回归下面四类问题：

1. 企业诊断是否稳定。
2. 风险拆层是否准确切换。
3. 决策建议是否能接在诊断之后继续问。
4. 模糊追问是否承接线程上下文，而不是丢对象或乱跳模式。

## 2. 核心样例

### 2.1 企业诊断起手

问题：`恒瑞医药当前最值得关注的经营问题是什么？`

预期：

1. 命中 `company_diagnosis`。
2. 返回企业诊断类标题和经营指标。
3. 不应直接跳到风险预测。

### 2.2 风险拆层

问题：`把恒瑞医药的风险拆成财务、经营、行业三层`

预期：

1. 命中 `company_risk_forecast`。
2. 返回风险等级、监测项或模型概率。
3. 推荐追问继续围绕风险或动作建议。

### 2.3 诊断后继续要建议

问题链：

1. `恒瑞医药当前最值得关注的经营问题是什么？`
2. `给出动作建议`

预期：

1. 第二轮能承接同一企业。
2. 第二轮不应回退到全局概览。
3. 应转到决策建议或明确的经营动作输出。

### 2.4 模糊追问承接

问题链：

1. `分析迈瑞医疗`
2. `展开讲讲`

预期：

1. 第二轮继续沿用线程焦点企业。
2. 若用户没有显式换任务，优先承接上一轮任务模式。
3. 线程不应丢成默认引导。

## 3. 页面体验样例

### 3.1 输入区位置

预期：

1. 输入框紧贴消息流底部。
2. 不会被结果卡片推到很远的位置。
3. 发送按钮始终在同一视线范围内。

### 3.2 结果编排

预期：

1. 结果区以内联折叠块展示，不要飘在内容上方。
2. 本轮结论、执行步骤、推荐追问的层级清晰。
3. 不出现浅底浅字导致不可读。

## 4. 赛题对齐检查

每轮回归都要问自己三件事：

1. 这一改动有没有让“智能体主导的企业分析”更明确。
2. 这一改动有没有让“来源追溯与证据链”更容易表达。
3. 这一改动有没有减少与赛题无关的页面噪音。
